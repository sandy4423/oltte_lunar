## 3. 픽업 주문 설정

> **코드 위치**: `src/lib/constants.ts` → `PICKUP_CONFIG`, `PICKUP_AVAILABLE_DATES`, `PICKUP_TIME_SLOTS`, `PICKUP_EARLY_CLOSE_DATES`

### 기본 설정

```typescript
export const PICKUP_APT_CODE = 'PICKUP';

export const PICKUP_CONFIG = {
  name: '픽업주문',
  deliveryDate: '2026-02-15',       // 픽업 가능 마지막 날짜
  cutoffAt: '2026-02-14T23:00:00+09:00', // 주문 마감
};
```

### 픽업 가능 날짜 (2026 설)

2/6(금) ~ 2/15(일), 총 10일

```
2026-02-06 ~ 2026-02-15
```

### 픽업 가능 시간

매일 09:00 ~ 21:00, 1시간 단위

```
09:00, 10:00, 11:00, 12:00, 13:00, 14:00,
15:00, 16:00, 17:00, 18:00, 19:00, 20:00, 21:00
```

### 조기 마감

| 날짜 | 요일 | 마지막 픽업 시간 | 사유 |
|---|---|---|---|
| 2026-02-14 | 토요일 | 13:00 | 오후 일찍 마감 |

### 픽업 할인 정책

- 기본 픽업 할인: **3,000원** (`PICKUP_DISCOUNT`)
- 픽업 할인 적용 최소 주문금액: **30,000원** (`PICKUP_DISCOUNT_THRESHOLD`)
- 픽업 최소 주문금액: **10,000원** (`PICKUP_MIN_ORDER_AMOUNT`)
- 단골톡방 할인: **1,000원** 추가 (`DANGOL_DISCOUNT`, `?source=dangol` 유입 시)

### 픽업 장소

```
e편한세상송도 후문상가 안쪽. 컴포즈 옆 (랜드마크로 113)
인천광역시 연수구 랜드마크로 113 e편한세상송도 후문상가 제114호 일부호
```

## 4. 상품 구성 및 가격

> **코드 위치**: `src/lib/constants.ts` → `PRODUCTS`

| SKU | 상품명 | 설명 | 가격 | 비고 |
|---|---|---|---|---|
| `ricecake_1kg` | 떡국떡 | 1kg | 10,000원 | 무료배송 대상 |
| `broth_1200ml` | 양지육수 | 1200ml | 5,000원 | 무료배송 제외 |
| `meat` | 고기만두 | 1팩 8알 | 10,000원 | 무료배송 대상 |
| `kimchi` | 김치만두 | 1팩 8알 (매콤) | 10,000원 | 무료배송 대상 |
| `half` | 반반만두 | 고기4 + 김치4 | 10,000원 | 무료배송 대상 |

### 주문 최소 수량

- **최소 3팩 이상** 주문 필수 (`MIN_ORDER_QUANTITY = 3`)
- 픽업의 경우 최소 주문금액 10,000원

### 무료배송 기준

- `FREE_SHIPPING_ELIGIBLE_SKUS`: `meat`, `kimchi`, `half`, `ricecake_1kg` 해당 품목이 기준 수량 이상이면 배송비 없음
- 양지육수(`broth_1200ml`)는 무료배송 카운트에서 제외

## 5. 마감 정책 설명

### 정상 흐름

```
주문 접수 → 가상계좌/카드 결제 → 배송
```

### 주문 상태(status) 목록

| 상태 | 의미 |
|---|---|
| `CREATED` | 주문 생성 (결제 전) |
| `WAITING_FOR_DEPOSIT` | 가상계좌 발급됨, 입금 대기 중 |
| `PAID` | 결제 완료 (카드 또는 가상계좌 입금) |
| `LATE_DEPOSIT` | 마감 이후 입금 확인됨 (특이 케이스) |
| `CANCEL_REQUESTED` | 취소 요청됨, 환불 계좌 대기 |
| `CANCELLED` | 취소 완료 |
| `AUTO_CANCELED` | 자동 취소 (미입금 기간 초과 등) |
| `OUT_FOR_DELIVERY` | 배송 출발 |
| `DELIVERED` | 배송 완료 |
| `REFUND_PROCESSING` | 환불 처리 중 |
| `REFUNDED` | 환불 완료 |

### 시간별 주문 가능 여부

```
주문 마감일 이전 (cutoffAt 전)
  → 정상 주문 가능

마감일 ~ 배송일 06:00
  → "추가 주문" 팝업 표시, 주문은 여전히 가능
  → 배송 가능 여부는 상황에 따라 수동 결정

배송일 06:00 이후
  → "픽업만 가능" 팝업 표시
  → 해당 단지 배송 주문 불가 (픽업 페이지로 유도)
```

## 6. 팝업 시나리오 전체 기록

> 주문 페이지(`/order?apt=코드`) 진입 시 시간대에 따라 팝업이 자동으로 결정됨
> **코드 위치**: `src/hooks/useOrderPopups.ts`, `src/app/order/page.tsx`

### 팝업 우선순위 로직 (`useOrderPopups.ts`)

```
1순위: 배송일 06:00 이후       → pickupOnly (픽업 전용 팝업)
2순위: cutoffAt 지남 (배송일 전) → extendedOrder (추가 주문 안내)
3순위: cutoffAt 당일 (날짜 기준) → zeroDayWarning (마감 임박)
4순위: 그 외 (마감 전 일반)    → welcome (환영 팝업)
```

---

### 팝업 1: welcome — 마감 전 환영 팝업

**노출 조건**: 마감일(cutoffAt) 이전이고, 마감 당일도 아닐 때

**제목**: (없음, 제목 없이 본문만)

**본문 내용**:
```
QR을 찍어주셔서 감사합니다!

저희는 e편한세상 후문에서
열심히 만두 빚고있는 올때만두입니다.

이번 설을 맞아,
정-말 정성스럽게 만두와 떡과 육수를
준비했습니다.

아래 주문페이지에서 주문 부탁드립니다 :)

맛있게 배달해드릴게요!

감사합니다.
          — 올때만두 드림
```

**UI 특징**:
- 배경 없이 흰 카드
- "만두와 떡과 육수" 굵은 글씨
- "올때만두" 브랜드 색상(주황)
- 클릭하면 닫힘 (`clickToClose={true}`)

### 팝업 2: zeroDayWarning — 마감 당일 임박 팝업

**노출 조건**: 오늘 날짜 == cutoffAt 날짜 (시간 무관, 날짜만 비교)

**제목**: `⏰ 주문 마감 임박!`

**본문 내용**:
```
QR을 찍어주셔서 감사합니다!

[단지명] 오늘 주문 마감입니다!

┌─────────────────────────┐
│ 현재 시각: M월 d일 HH:mm:ss │
│                         │
│       HH:mm에 마감!       │  ← 실시간 업데이트
│   ⏰ XX시간 XX분 남음      │  ← 실시간 카운트다운
└─────────────────────────┘

마감 시간 이후에는
주문이 불가능합니다.

지금 바로 주문하세요! 🥟
```

**UI 특징**:
- 마감 시간 및 카운트다운을 주황색 배경 박스로 강조
- 단지명은 `getApartmentFullName(apartment)` 함수 결과 (동 범위 포함)
- 카운트다운은 1초마다 실시간 업데이트

### 팝업 3: extendedOrder — 마감 후 추가 주문 안내 팝업

**노출 조건**: cutoffAt 이후 && 배송일 당일 06:00 이전

**제목**: `📢 추가 주문 안내`

**본문 내용**:
```

주문 마감일이 지났지만,

│                            │
│    M월 d일 (요일) 06:00에 마감! │

└────────────────────────────┘
```

- "배송일 새벽 6시까지 추가주문 받습니다!" 2xl 굵은 글씨, 브랜드 다크 색상

- 카운트다운 기준: `배송일 당일 00:00` + 6시간 = 배송일 06:00

## 7. 팝업 변경 이력

### v1 (초기 런칭, 1월 말 ~ 2월 초)

- `welcome` 팝업만 존재
- 마감 당일 별도 처리 없음
- 마감 후에는 단순히 주문 불가 상태 표시

### v2 (운영 중 변경 1 — 마감 임박 대응)

- `zeroDayWarning` 팝업 추가
- 마감 당일 실시간 카운트다운 기능 추가
- 배경: 일부 고객들이 마감 당일 QR 스캔 후 "주문이 안 된다"고 혼동하는 경우 발생

### v3 (운영 중 변경 2 — 마감 연장 결정)

- `extendedOrder` 팝업 추가
- 배경: 마감 후에도 주문 요청이 계속 들어와서, 배송 전날 밤까지 운영하기로 결정
- 기존에는 마감 후 접속 시 그냥 "마감됨" 표시였으나, 적극적으로 추가 주문 유도
- **이 팝업이 추가된 후 주문량이 유의미하게 증가함**

### v4 (배송일 당일 처리 추가)

- `pickupOnly` 팝업 추가
- 배경: 배송일 당일 06:00 이후에도 QR을 스캔하는 고객 존재 → 픽업으로 유도
- 이전에는 배송일 이후 접속 시 오류 없이 주문 가능 상태여서 혼선 발생

## 11. 운영 중 발생한 이슈 및 대처법

### 이슈 1: 전단지 마감일과 실제 마감일 불일치

**상황**: 전단지에 D-4로 인쇄했으나 실제로는 D-1까지 받기로 결정  
**대처**: `originalCutoffAt` 필드를 전단지 날짜로 보존하고, `cutoffAt`을 실제 운영 날짜로 분리  
**교훈**: 다음엔 전단지에 마감일을 D-2 정도로 더 여유 있게 인쇄할 것

### 이슈 2: 마감 후에도 주문 요청 폭주

**상황**: 마감 후 QR 스캔으로 들어온 고객들이 "왜 주문이 안 되냐"고 문의  
**대처**: `extendedOrder` 팝업 추가, 배송일 전날까지 추가 주문 수용  
**교훈**: 처음부터 D-1 마감으로 전단지 제작 추천

### 이슈 3: 카드사 심사 기간 중 휴대폰 인증 없음

**상황**: 토스페이먼츠 카드 결제 연동을 위한 카드사 심사 중, 심사관이 직접 주문을 테스트  
**대처**: `SKIP_PHONE_VERIFICATION = true`로 설정해 SMS 없이 주문 가능하게 함  
**교훈**: 카드사 심사 전에 이 플래그를 켜두고, 심사 완료 후 `false`로 되돌릴 것

### 이슈 4: 가상계좌만 있었을 때 카드 결제 요청

**상황**: 초기 런칭 시 가상계좌 입금만 지원했는데, 카드 결제 요청 다수  
**대처**: 토스페이먼츠 카드 결제 연동 추가, 결제 수단 선택 페이지 구현  
**교훈**: 처음부터 카드 + 가상계좌 선택 UI를 준비해 둘 것

### 이슈 5: 주문 취소 방법 없어 고객 문의 급증

**상황**: 취소는 전화로만 가능하여 운영자 부담 가중  
**대처**: `/my-orders` 페이지에 온라인 주문 취소 기능 추가  
**교훈**: 다음 명절엔 취소 기능 포함한 채로 런칭

### 이슈 6: 비회원 주문 명시 부재로 카드사 심사 지적

**상황**: 카드사 심사에서 "비회원 주문 명시가 필요하다"는 피드백  
**대처**: 주문 페이지 상단에 "회원가입 없이 주문 가능 (비회원 주문)" 안내 추가  
**교훈**: 처음부터 비회원 주문 명시 필수
