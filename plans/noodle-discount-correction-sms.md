# 칼국수 할인 누락 고객 5명 정정 SMS 발송

> 요청일: 2026-04-12
> 지시: "칼국수 할인 오류났던 5분에게 (성하경 제외) 문자로 안내드리자. 계획 세워."

## 배경

- 2026-04-10 ~ 04-11 사이 단골톡방(`source='dangol'`) 주문 중, `useOrderSubmit.ts` 서버측 할인 계산에 **칼국수 500원 할인 로직이 누락**되어 있어 `dangol_discount`가 전골 할인(2,000원)만 반영됨.
- 커밋 `419cb41`에서 `pricing.ts`의 `calculateDangolDiscount` 공유 함수로 리팩토링하면서 수정 완료.
- 그러나 **수정 이전에 이미 결제된 주문 6건은 고객이 500원씩 더 지불한 상태**. DB 보정은 불가(Toss 결제 완료 건).
- 6건 중 1건(성하경)은 사장님 아내분이라 제외 → **5명**에게 정정 안내 + 현금 환급 약속 SMS 발송.

## SMS 본문안

```
[올때만두]
안녕하세요 {이름}님 올때만두입니다.

단골톡방 할인 계산에 시스템 오류가 있어
칼국수 500원 할인이 누락되어 결제되었습니다.
세심하지 못해 정말 죄송합니다.

다음 매장 방문 시 현금 500원으로
돌려드리겠습니다. 영수증 없이 이 문자만
보여주시면 됩니다.

불편 드려 죄송합니다.
올때만두 사장 드림
문의: 010-5877-4424
```

- **Why:** 과한 사과보다 원인 명시 + 구체적 환급 방법(매장·현금·문자 확인) 제시. 사장님이 직접 작성한 톤.
- 누락 금액은 전원 500원이라 템플릿에 하드코딩 가능 (개인화는 이름만).
- 총 5건 × 약 100자 → 단문 SMS 범위 내.

## 구현 방안 (확정: 옵션 C — 스크립트)

**기존 패턴 재사용**: `scripts/send-ricecake-issue-sms.ts`를 원형으로 **일회성 스크립트** 작성. Next 빌드/배포 불필요, 로컬에서 직접 `npx tsx` 실행.

### 파일 추가
`scripts/send-noodle-refund.ts`
- 상단에 `.env.local` 직접 로드 (기존 ricecake 스크립트와 동일 패턴)
- `SOLAPI_SENDER_NUMBER`를 등록된 발신번호(`01025924423`)로 강제 설정
- `RECIPIENTS` 상수에 5명 이름·전화번호 하드코딩 (Supabase 재조회 없음 — 명단 확정됨)
- 본문 템플릿에 `${name}` 치환
- `--test` 플래그: 관리자 번호(01035804423)로만 1건 발송
- 실제 모드: 5명 순차 발송, 각 건 사이 1초 딜레이 (Solapi rate limit)
- 출력: 각 건 성공/실패 라인 + 최종 요약 (성공 N, 실패 N, 총 5)

### 실행 방식 (확정)
- **사장님이 대화창에서 "발송해" 지시 → Claude가 Bash로 `npx tsx scripts/send-noodle-refund.ts` 직접 실행**
- 테스트 먼저: `npx tsx scripts/send-noodle-refund.ts --test` (사장님 번호로 1건)
- 테스트 확인 OK 후 → 실제 발송
- 발송 로그는 대화창에 그대로 노출됨 (감사 흔적)
- 배포 불필요, route 파일 생성 불필요 → 제거 커밋도 불필요
